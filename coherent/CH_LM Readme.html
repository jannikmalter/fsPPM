Refer to the LabVIEW Help for more information about software requirements. You access the LabVIEW Help by selecting Help»Search the LabVIEW Help.

3. Known Issues

To report issues or provide feedback about this instrument driver, please send an email to instrument.drivers@ni.com or support@BrandNewTechnologies.de.

  • The ActiveX Control and or LabMax meter are causing timeouts in rare cases.
  • LabMax application can lock up when using LV 7.1. This problem seems to be not existent with LV 8.2.
    Lock up is unpredictable due to unknown reason. Moving windows or performing other CPU consuming task lead to higher probability for lock up. You need to restart the application/LabVIEW and may perform a LabMax power cycle to restart.
  • Data Delivery Mode = STREAM is recommended for USB only. In case of GBIP or RS232 communication mode it works for low rates, high rates will cause data queue pileup in the meter and data will become displayed with delay due to slow readout.

4. Getting Started ...

Open the CH_LM VI Tree.vi. In the upper left corner of the block diagram you find the CH_LM ActiveX Control.vi. This VI must be started first and prior to all other VIs of this driver. You can start it manually or call it from your application VI. You have to specify the communication mode. Only one communication mode can be selected at one time. Mixed communication modes are not supported by the current version of the LabMax ActiveX Control. You can stop the VI from the front panel or via occurrence in an application.

You have to call the CH_LM VI Initialize.vi for every meter you want to use. You need to supply the address, GPIB address or COM port number, or, in case of USB, meter index or serial number. You have to call the CH_LM Close.vi for every meter to disconnect.

You can view the connected meters with serial number and index in: CH_LM Utility Monitor Meter List.vi.

Refer also to the Example VIs to get started.

5. Demo Application ...

Open the CH_LM VI Tree.vi. In the upper right corner of the block diagram you find the CH_LM App Main.vi which starts CH_LM ActiveX Control.vi automatically, so, it must not be started before. You will be prompted to specify the communication mode. Only one communication mode can be selected at one time. Mixed communication modes are not supported by the current version of the LabMax ActiveX Control. In case of GPIB and RS232 communication you need to sepcify the LabMax meters to use via Menu->Settings->Interface.... Then you are prompted to select the LabMax meters to be used by the current frontpanel. You can do also change your selection later via Menu->File->Select Devices or by clicking the button in the upper left corner. If you want to read data from multiple selected meters in synchronous mode you have to check the menu option Menu->Settings->DAQ->Synchronous. Use Menu->Help->Show Context Help or press CTRL+H to open the context help window and point with the mouse to the desired control/indicator.
